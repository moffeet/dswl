👥 客户管理 API
GET /api/customers
获取客户列表（分页）

Query 参数：

参数名 类型 必填 说明 示例
page number 否 页码，默认 1 1
limit number 否 每页数量，默认 10 10

响应示例：

{ "code": 0, "message": "获取成功", "data": [ { "id": 1, "customerNumber": "C001", "customerName": "北京华领科技有限公司", "customerAddress": "北京市朝阳区建国门外大街123号", "updateTime": "2024-01-15T10:30:00.000Z", "updateBy": "系统" } ], "total": 1, "page": 1, "limit": 10 }
GET /api/customers/search
搜索客户

Query 参数：

参数名 类型 必填 说明 示例
customerCode string 否 客户编号（模糊搜索） C001
customerName string 否 客户名（模糊搜索） 华领
customerAddress string 否 客户地址（模糊搜索） 北京

响应示例：

{ "code": 0, "message": "搜索成功", "data": [...], "total": 5 }
GET /api/customers/{id}
获取客户详情

路径参数：

参数名 类型 必填 说明 示例
id number 是 客户ID 1
POST /api/customers
创建客户

请求体参数：

参数名 类型 必填 说明 示例
customerName string 是 客户名称（最长100字符） 李果果公司
customerAddress string 否 客户地址（最长200字符） 广州市天河北28号

请求示例：

{ "customerName": "李果果公司", "customerAddress": "广州市天河北28号" }

响应示例：

{ "code": 0, "message": "创建成功", "data": { "id": 8, "customerNumber": "C008", "customerName": "李果果公司", "customerAddress": "广州市天河北28号", "updateTime": "2024-01-15T10:30:00.000Z", "updateBy": "管理员" } }
PATCH /api/customers/{id}
更新客户信息

路径参数：

参数名 类型 必填 说明 示例
id number 是 客户ID 1

请求体参数：

参数名 类型 必填 说明 示例
customerName string 否 客户名称 更新后的客户名
customerAddress string 否 客户地址 更新后的地址
DELETE /api/customers/{id}
删除客户

路径参数：

参数名 类型 必填 说明 示例
id number 是 客户ID 1

📋 统一响应格式

所有 API 响应都采用统一的 JSON 格式：

{ "code": 0, // 状态码：0=成功，其他=失败 "message": "操作成功", // 提示信息 "data": {}, // 响应数据（可选） "total": 100, // 总数（列表接口） "page": 1, // 当前页（列表接口） "limit": 10, // 每页数量（列表接口） "error": "错误详情" // 错误信息（失败时） }

❌ 常用错误码

错误码 说明 示例
0 操作成功 { "code": 0, "message": "操作成功" }
400 请求参数错误 { "code": 400, "message": "参数验证失败" }
404 资源不存在 { "code": 404, "message": "客户不存在" }
500 服务器内部错误 { "code": 500, "message": "服务器错误" }

🔧 开发提示

  • 客户编号 (customerNumber) 由系统自动生成，创建时不需要传递
  • 所有搜索接口都支持模糊匹配
  • 分页查询的 page 从 1 开始
  • 删除操作会返回被删除的记录信息
  • 建议在请求失败时检查 error 字段获取详细错误信息
